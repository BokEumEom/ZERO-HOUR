# LEARNINGS

개발 이터레이션에서 실제로 발견된 문제 패턴과 해결책의 증류본.
새 이터레이션에서 같은 함정을 밟기 전에 먼저 읽을 것.

## 1. 도메인 로직 — 시간·상태

### 자정 롤오버: "현재 날짜"를 캐시하면 언젠가 틀린다
- **문제**: `recs.today`를 부팅 시 1회 캐시 → 페이지를 자정 넘겨 열어두면 새 날짜의 런이 어제 키에 저장돼 streak이 끊김. 결정적으로, **새로 추가한 카운트다운 기능이 사용자를 자정까지 대기하도록 유도**해 잠재 버그를 현실 버그로 격상시킴.
- **해결**: ① 기록은 "지금 날짜"가 아니라 **런의 시드 날짜**(`res.seedStr`)로 저장 — 데이터가 자신의 출처를 들고 다니게 한다. ② 경계 이벤트(런 시작, 게임 오버, 카운트다운 0초)마다 `syncToday()`로 캐시 갱신.
- **교훈**: 기능 추가는 기존 가정의 노출 빈도를 바꾼다. "그 시나리오는 드물다"는 가정은 그 시나리오를 유도하는 기능이 들어오는 순간 무효.

### 일시정지의 진짜 어려움은 멈춤이 아니라 입력 누수
- **문제 패턴**: ① blur로 자동 일시정지되면 keyup이 영원히 안 옴 → 키가 눌린 채 고정. ② 오버레이 위에서 누른 키가 재개 직후 반영. ③ 조이스틱 드래그 중 일시정지 시 축이 남음. ④ 메뉴에서 blur된 키가 **다음 런**까지 살아남음.
- **해결**: 상태 전이 지점마다 입력을 리셋 — `pause()`, `resume()`, **그리고 `start()`**에서 `resetKeys()` + 축 0. 전이마다 "이 시점에 살아있으면 안 되는 입력은 무엇인가"를 물을 것.

### 중도 포기(QUIT)는 "기록 경로를 안 타는 것"으로 구현
- `endGame()`을 경유하지 않고 `toMenu()`로 직행하면 onGameOver·저장이 자연히 생략됨. 별도 "discard 플래그"보다 경로 분리가 단순하고 테스트하기 쉬움 (spy 호출 0 + IndexedDB 불변 assert).

## 2. 헤드리스 브라우저 테스트

### 가상 시간(virtual-time-budget)에서 rAF는 굶는다
- **증상**: 타이머(setTimeout/setInterval)는 빨리 감기되는데 requestAnimationFrame 루프는 거의 안 돎 → 게임이 ready에서 영원히 정지.
- **해결**: 테스트 하니스가 `G.update(dt)`를 직접 호출해 시뮬을 수동 스텝. 엔진을 rAF에서 분리 가능하게 설계(`G.update` 공개)한 것이 테스트 가능성의 핵심이었음.

### 정적 서버의 cleanUrls가 쿼리스트링을 삼킨다
- `npx serve`는 `/page.html?s=x` 리다이렉트 과정에서 쿼리를 유실 → 하니스 시나리오 라우팅이 조용히 기본값으로 떨어짐. **해시(`#scenario`)는 리다이렉트에도 살아남는다.**
- 디버깅법: 상태를 화면 텍스트로 렌더링하고 스크린샷으로 읽기 (`--dump-dom`은 환경에 따라 무출력).

### 자명하게 통과하는 어서션은 통과가 아니다
- "브레이크다운 합 == 점수"가 0 == 0으로 통과, "스파크라인 not blank"가 평평한 기준선으로 통과. **assert가 비자명한 입력 위에서 도는지 출력 로그로 확인할 것** (`score=348` 같은 디테일을 리포트에 포함). 결정적 픽스처 주입(플레이어 위치에 크리스털 강제 스폰)으로 flake 없이 비자명성 확보.

## 3. 테스트 설계

### IIFE 게임 코드도 의존성 0으로 단위 테스트 가능
- `vm.createContext` 샌드박스에 `window = sandbox` + 리스너 캡처 + `SY.audio`는 Proxy noop. IndexedDB는 사용하는 API 표면만(open/transaction/get/put) ~50줄로 스텁. 시계는 `Date` 서브클래스로 동결 — 월/연 경계 픽스처가 공짜.
- **함정**: vm realm의 Array는 호스트 `deepStrictEqual`에서 프로토타입 불일치로 실패 → `Array.from()`으로 호스트 realm 복사 후 비교.

### 컨벤션은 tripwire 테스트로 강제해야 산다
- "게임플레이 랜덤은 시드 RNG만", "innerHTML 싱크 추가 금지", "인라인 style 금지" 같은 규칙은 문서만으로는 자신도 어긴다(이번에 인라인 style 3건 자가 위반). **개수 베이스라인 핀**(Math.random ≤14, innerHTML 싱크 == 3, inline style ≤6)으로 위반 시 테스트가 강제 리뷰를 발동하게 할 것.

### 생성 산출물 동기화는 "커밋 독립적" 검사로
- `git diff --exit-code`는 커밋 전엔 항상 실패하는 잘못된 검사. **임시 경로로 fresh build → 해시 비교**가 올바른 "재현 가능 + 동기화" 검사. 훅(직접 수정 차단 + stale 리마인드)과 합쳐 3중 방어.

### 독립 Verifier는 all-green 뒤의 버그를 찾는다
- 테스트 전부 통과 상태에서 적대적 리뷰 에이전트가 실질 버그(자정 롤오버)와 규칙 위반(인라인 style)을 적발. 구현자와 같은 컨텍스트의 셀프 리뷰는 같은 맹점을 공유함 — **"기준을 위반하는 방법을 적극적으로 찾아라"는 프롬프트가 유효**했음.

## 4. UI 패턴

### CSS 기본값이 `display:none`이면 JS의 `''` 복원이 안 통한다
- `el.style.display = ''`는 인라인 제거 → CSS 기본값(none)으로 폴백 → 영원히 안 보임. 같은 버그를 `#btn-pause`와 `#menu-week`에서 두 번 만남. **숨김 기본 + JS 토글 요소는 표시 값('block')을 명시**하거나, 클래스 토글로 통일할 것.

### 지연 표시되는 화면의 단축키는 phase가 아니라 DOM visibility로 가드
- 게임 오버 화면은 phase 전환 650ms 후 표시 → phase 기준이면 화면이 뜨기 전에 R/Enter가 동작. `classList.contains('visible')` 체크가 정답. keydown 분기 순서(over > howto > paused > playing)로 Esc의 다중 역할 충돌 해소.

## 5. Windows PowerShell 5.1 (이 프로젝트의 셸 환경)

| 함정 | 해결 |
|---|---|
| `Get-Content`가 BOM 없는 UTF-8을 ANSI로 읽음 → **한글 파일 통째로 깨짐** (실제 사고, git으로 복원) | 비ASCII 파일은 반드시 `[IO.File]::ReadAllText($p, [Text.Encoding]::UTF8)` / `WriteAllText` |
| 내부 따옴표 포함 멀티라인 인자가 native 명령에서 깨짐 (`git commit -m` 실패) | 메시지를 파일로 쓰고 `git commit -F` |
| `Process.WaitForExit(timeout)` 후 `ExitCode`가 null | `HasExited` 확인 후 **인자 없는** `WaitForExit()` 한 번 더 |
| `$profile`은 자동 변수 — 대입 시 부작용 | 변수명 회피 (`$edgeProfile`) |
| `Out-File`/`Set-Content` 기본 인코딩이 UTF-16/BOM | 다른 도구가 읽을 파일은 인코딩 명시 |

## 6. 프로세스

- **시드 RNG 게임의 점수 집계는 버킷 분해 + "합 ≡ 총점" 불변식**으로 설계하면 property 테스트 한 방으로 모든 가산 경로를 커버한다 (×2 배율도 버킷 안에서 적용해야 불변식 유지).
- **결정적 시뮬은 E2E 재현성의 토대**: 날짜 시드 + 수동 스텝 → 같은 테스트가 항상 같은 점수(348)를 산출. flaky 어서션이 원천 차단됨.
- 훅 리마인더(소스 변경 → standalone stale)는 시끄럽지만 실제로 재빌드 누락을 한 번도 허용하지 않았음 — 노이즈보다 가드 가치가 컸다.
