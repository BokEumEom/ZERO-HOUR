# ADR-0002: 시드 RNG로 데일리 챌린지 공정성 보장

- **상태**: 적용됨 (이전부터 존재, 2026-06-11 tripwire 테스트로 강제화)

## Context

데일리 챌린지는 "오늘은 전 세계가 같은 맵"이 핵심 가치다. 같은 날짜 시드
(`daily-YYYY-MM-DD` UTC → xmur3 해시 → mulberry32)에서 모든 플레이어가 동일한
크리스털·적·파워업 배치를 받아야 점수 경쟁이 공정하다. `Math.random()` 하나가
스폰 로직에 섞이는 순간 이 보장이 조용히 깨진다.

## Decision

- **게임플레이에 영향을 주는 모든 난수는 런 상태의 `s.rng()`** (스폰 위치/개수, 드랍 확률, 파워업 가방, 적 진입 방향).
- **`Math.random()`은 코스메틱 전용** (파티클, 화면 흔들림, 보스 사망 연출, 오디오 노이즈) + 프리플레이 시드 문자열.
- 규칙은 문서가 아니라 **tripwire 테스트로 강제** (`test/unit/static.test.mjs`):
  - `Math.random()` 호출 수 베이스라인 핀 (≤14) — 신규 호출은 테스트 실패로 공정성 리뷰 강제
  - 핵심 드랍 사이트(`drops = 4 + s.rng()`, `s.rng() < 0.45`)의 시드 사용을 grep assert
  - `makeRng` 결정성(같은 시드 → 같은 수열) 단위 테스트

## Consequences

- (+) 공정성 회귀가 코드 리뷰가 아닌 CI에서 잡힘
- (−) 개수 핀은 무딘 가드(위치 이동은 못 잡음) — 의도된 트레이드오프: 정밀 추적보다 "변경 시 강제 리뷰" 신호가 목적
